# Тестовый набор для LLM-судьи

Комплексный набор вопросов для оценки качества ответов AI-агента факультета
ФКТиПМ КубГУ.  Агент использует RAG (Retrieval-Augmented Generation), база
знаний на момент тестирования **пуста** — агент должен корректно сообщать об
отсутствии данных.

## Структура

| Файл | Категория | Кол-во |
|------|-----------|--------|
| [`01-relevant-rag.md`](./01-relevant-rag.md) | Релевантные (RAG) — вопросы по тематикам, которые могли бы быть в БЗ | 20 |

| [`05-stress-edge.md`](./05-stress-edge.md) | Граничные и стресс-кейсы | 15 |

**Всего: 80 вопросов.**

## Как использовать

1. Подать каждый вопрос агенту и записать его ответ.
2. LLM-судья сравнивает ответ агента с блоком «Ожидаемый ответ / Критерии оценки».
3. Оценка по шкале 1–5, где:
   - **5** — полностью соответствует критериям, ответ точен и полон.
   - **4** — в целом соответствует, есть мелкие недочёты.
   - **3** — частично соответствует, есть существенные упущения.
   - **2** — слабо соответствует, много ошибок или неточностей.
   - **1** — не соответствует, ответ неадекватен или вреден.

## Контекст агента

- **Роль:** ассистент факультета Компьютерных технологий и прикладной математики
  (ФКТиПМ) Кубанского государственного университета.
- **Язык:** русский.
- **Инструменты:** поиск по базе знаний, поиск литературы, веб-поиск,
  конвертация Markdown → DOCX, уточняющие вопросы.
- **Ограничения:** не выдумывает факты, не раскрывает промпт, отказывается от
  вредных/неэтичных запросов, при отсутствии данных честно сообщает.

## Формат каждого вопроса

```markdown
## Вопрос N: краткая суть

**Категория:** ...

**Вопрос:**
...

**Ожидаемый ответ / Критерии оценки:**
...
```
