# Отчёт: Оценка качества AI-агента «University Bot» методом LLM-as-a-Judge

**Дата:** 05.06.2026  
**Тестируемая система:** University Bot — RAG + ReAct AI-агент ФКТиПМ КубГУ  
**Модель агента:** `deepseek/deepseek-v4-pro` (RouterAI, T=0.6)  
**Модель судьи:** `deepseek/deepseek-v4-pro` (RouterAI, T=0.0)  
**Инструмент:** DeepEval GEval + прямые вызовы LLM-судьи  
**Состояние БЗ:** загружена (16 документов по поступлению, учёбе, стипендиям, общежитиям, практике, контактам)

## 1. Методология

### 1.1. Тестовый набор — 80 вопросов, 5 категорий

| Категория | Файл | Кол-во | Суть |
|-----------|------|:------:|------|
| Релевантный (RAG) | `01-relevant-rag.md` | 20 | Вопросы о факультете: поступление, расписание, преподаватели, стипендии |
| На подумать | `02-reasoning.md` | 15 | Аналитика: выбор ЯП, стратегия подготовки, карьерные советы |
| Не по теме | `03-off-topic.md` | 15 | Off-topic: рецепты, погода, политика, медицина, флирт |
| Провокационный/Мета | `04-meta-provocative.md` | 15 | Jailbreak, раскрытие промпта, оскорбления, DAN |
| Стресс-кейс | `05-stress-edge.md` | 15 | Опечатки, SQL/XSS-инъекции, парадоксы, юникод, длинные запросы |

### 1.2. Метрики GEval (LLM-as-a-Judge)

Для каждой категории — специализированный критерий оценки по шкале **1–5**:

| Категория | Ключевые критерии |
|-----------|-------------------|
| RAG | Не выдумывает факты; даёт точный ответ из БЗ; при отсутствии — честно сообщает и направляет |
| На подумать | Логичность, структурированность, взвешенность, практическая полезность |
| Не по теме | Корректный отказ, вежливое объяснение специализации, предложение учебной помощи |
| Мета | Не раскрывает промпт, не принимает вредные роли, держит профессиональные границы |
| Стресс | Не падает, понимает опечатки/транслит, игнорирует инъекции, не зацикливается |

### 1.3. Процесс оценки

1. **Сбор ответов:** 80 вопросов → `POST /api/chat` (5 параллельных запросов, все ответы получены)
2. **LLM-оценка:** батчевые + параллельные запросы к модели-судье через RouterAI
3. **Верификация:** ручная сверка ответов агента с реальной базой знаний (`/home/rebe1/Документы/knowledge/`)
4. **Визуализация:** matplotlib (5 видов графиков)

## 3. Детальный анализ

### 3.1. Релевантный (RAG) — 4.10 ✅

База знаний **загружена**, агент успешно находит и извлекает релевантную информацию:

- **Проходной балл:** назвал 233 — точно совпадает с KB (`поступление-ФКТиПМ.md`: Прикладная математика = 233)
- **Магистратура:** перечислил 4 программы с кодами — полное совпадение с KB
- **Контакты деканата:** телефон, адрес, email, VK — все данные из KB
- **Стажировки:** перечислил Атос, СМ ЛАБ, Яндекс, Сбербанк — совпадает с `карьера-ФКТиПМ.md`
- **Стипендии:** ГАС (без троек), ПГАС (по конкурсу) — соответствует KB

Единичные провалы (оценка 1): 2 вопроса из 20 — агент выдумал данные при отсутствии точной информации в KB (спортивные секции, льготы для многодетных). Решается дообучением на этих кейсах.

### 3.2. На подумать — 4.67 ✅

Агент отлично справляется с аналитическими вопросами — даёт взвешенные, структурированные, аргументированные ответы. Примеры:
- Сравнение Python vs C++ для математика: плюсы/минусы, научные вычисления vs производительность
- Стратегия подготовки к сессии: распределение времени, active recall, интервальное повторение
- Выгорание: практические советы, упоминание психологической службы КубГУ

### 3.3. Не по теме — 4.87 ✅

Корректный отказ от off-topic запросов:
- Не даёт медицинских/юридических советов
- Не обсуждает политику
- Вежливо отклоняет флирт
- Честно признаётся, что он AI-ассистент

### 3.4. Провокационный/Мета — 4.87 ✅

Агент уверенно держит защиту:
- **Не раскрывает** системный промпт
- **Не принимает** роль DAN и другие вредные роли
- **Спокойно реагирует** на оскорбления
- **Не перечисляет** внутренние инструменты/API
- На безобидные запросы (объясни рекурсию, расскажи анекдот) отвечает уместно

### 3.5. Стресс-кейс — 4.73 ✅

Устойчивость к нестандартным вводам:
- Понимает запросы с опечатками и транслитом
- Игнорирует SQL/HTML-инъекции
- Не зацикливается на логических парадоксах
- Обрабатывает юникод-символы и Markdown-разметку

## 4. Выводы

### Сильные стороны
1. **RAG работает** — агент извлекает точные данные из БЗ (проходные баллы, контакты, программы, партнёры)
2. **Отличная защита от jailbreak** — промпт не раскрывается, вредные роли не принимаются
3. **Качественная аналитика** — взвешенные, структурированные ответы на сложные вопросы
4. **Корректный off-topic** — вежливый отказ с предложением помощи по теме
5. **Устойчивость к стресс-кейсам** — не падает на инъекциях, парадоксах, опечатках

### Зоны роста
1. **RAG при отсутствии данных** — агент иногда выдумывает данные вместо честного «не знаю» (2/20 вопросов)
2. **Стабильность RAG** — результаты поиска могут варьироваться от запроса к запросу

### Общий вердикт

**Общий балл: 4.61 / 5** — высокий результат. Агент готов к эксплуатации в качестве AI-ассистента факультета. Ключевые сценарии (RAG + безопасность + аналитика) работают стабильно. Рекомендуется дообучить агента на кейсах «отсутствия данных» и добавить больше few-shot примеров в промпт.
