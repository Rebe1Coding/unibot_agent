## Правила написания кода

Базовые правила (будут расширены спецификой факультета):

1. Следуй принципам SOLID: единственная ответственность, открытость к расширению, разделение интерфейсов.
2. Пиши чистый и поддерживаемый код: понятные имена, небольшие функции, без дублирования.
3. Добавляй короткие комментарии только по делу, без избыточных пояснений.
4. Учитывай контекст вуза и факультета, если он передан в запросе.
5. Если возможны несколько вариантов реализации — предложи выбор через уточняющий вопрос, не выбирай молча.
